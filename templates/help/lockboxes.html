What a lockbox record tracks

Each lockbox has a label, an auto-generated ID (like LBA001), its current and previous codes, an optional Supra ID, a storage location, a status, and (optionally) the property it's assigned to.

Adding a lockbox

  1. Go to Lockboxes → Add Lockbox.
  2. Enter a label (e.g. "LB-12") and the initial code — both are required.
  3. Save. To assign it to a property, open the new lockbox and use Edit — assignment is deliberately a separate step from creation.

Checking out / in

  • From the Lockboxes list, use the row's Check Out button, or use the Check In/Out page and scan/type the lockbox ID.
  • Checking out records who took it and why.
  • When you check a lockbox back in you can enter a new code — the old one is preserved as "previous code" so you can still get in if the change hasn't been made physically yet.
Tip: Use bulk operations — tick the checkboxes on several rows to update status or assignment in one go. Bulk delete is admin-only.

Codes

The current code is what opens the box today; previous code is kept automatically whenever the code changes. Code history matters after turnovers — if a contractor still has the old code, change the code and check the box back in with the new one.

Labels & QR codes

Open any lockbox and use Print Label to get a printable label with a scannable code — scanning it on the Check In/Out page pulls the item up instantly.

Exporting

The Export ▾ button on the Lockboxes list downloads the current inventory as CSV, Excel, or PDF.
