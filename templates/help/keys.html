How keys work in KBM

A key record represents all copies of one key. It tracks the total number of copies, how many are currently checked out, the key hook number where it hangs in your key box, an optional key code (the cut code), and the property/unit it opens.

Adding a key

  1. Go to Keys → Add Key.
  2. Enter a label, how many copies you have, and the hook number where it lives.
  3. Optionally link it to a property/unit — the address fills in automatically.
  4. Optionally set a master key if this key is part of a master system.

Checking copies out and in

  • Use the row's Check Out button or the Check In/Out page.
  • Choose how many copies, who's taking them (pick a contact or type a name), the purpose, and the expected return date.
  • Each checkout creates a record you can print a receipt for, and shows up under the key's active checkouts until it's returned.
  • Check-in asks how many copies came back — partial returns are fine.

Assigning vs. checking out

Check out is temporary (has an expected return date). Assign is long-term — e.g. a tenant or contractor keeps a copy indefinitely. Assigned copies are tracked separately so they don't show up as overdue.

A red dot next to a key means its total copies are at or below the low-keys threshold set in Settings — time to cut more copies. The Low Copy Report under Audits lists them all.

Auditing your key box

Use Audits to run a physical count of every hook and reconcile the system with reality — see the Audits help topic.
