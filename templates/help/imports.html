What you can import

Keys, lockboxes, signs, properties, property units, and smart locks can all be bulk-loaded from CSV or Excel (.csv, .xlsx, .xls) files. Each list page has its own Import button.

The import wizard

  1. Upload — pick your file. Need a starting point? Download the blank template from the upload page to see the expected columns.
  2. Map columns — tell KBM which of your spreadsheet's columns is the label, hook number, code, address, and so on. KBM guesses matches from your header names; correct anything it got wrong.
  3. Review — a preview of what will be created, with problems flagged before anything is saved: duplicates of existing records, rows missing required fields, and unknown property names (with fuzzy-match suggestions and the option to create the property on the fly).
  4. Process — the rows are imported, and you get a results summary of what was created and what was skipped.
Imports never overwrite existing records — duplicate rows are flagged in the review step so you can decide what to do.
Tip: keep one row per key type — the "total copies" column covers multiple physical copies of the same key.
