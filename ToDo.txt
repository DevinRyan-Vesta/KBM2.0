KBM 2.0 — Project To-Do / Roadmap
Last reviewed: 2026-07-02 (v2.1.0)

For the detailed history of what changed and when, see CHANGELOG.md.
(The old "reference information" fix log that used to live in this file is
covered by the changelog and git history.)

========================================
OUTSTANDING — HIGH PRIORITY
========================================

[ ] Rotate SECRET_KEY on every existing deployment. Old keys were committed
    to git history and must be treated as compromised. Production now refuses
    to boot without one (v2.1.0), but rotation on the VPS is a manual step.
    (Owner decision 2026-07-02: deferred — do at the next production deploy.)

[ ] Reduce the blast radius of the self-updater: the app container mounts
    the Docker socket (root-equivalent on the host) so the System Updates UI
    can rebuild containers. Consider moving the updater to a separate
    admin-only sidecar, or an explicit "maintenance mode" that is off by
    default. (The updater itself was overhauled and health-checked in
    v2.1.0 — this item is only about reducing its privileges.)

========================================
OUTSTANDING — MEDIUM PRIORITY
========================================

[ ] Report scheduling — email selected reports on a schedule
    (daily/weekly/monthly). The overdue-reminder cron
    (utilities/send_overdue_reminders.py) is a good pattern to extend.

[ ] Photo uploads for items and properties (smart locks already support
    photos with validation — reuse that implementation).

[ ] Duplicate detection on item creation (contacts got it in v2.1.0; keys/
    lockboxes/signs single-record forms could warn on matching labels too).

[ ] Loading states on slow actions (imports, exports, bulk operations).

========================================
OUTSTANDING — LONG TERM
========================================

[ ] Role-based permissions refinement — read-only role, checkout-only role.
[ ] Self-service PIN reset (email link) so admins don't have to reset PINs.
[ ] REST API with token auth for integrations.
[ ] Export column customization (choose which fields to include).
[ ] PDF export formatting improvements.
[ ] Report/export branding customization per company account.
[ ] Better mobile ergonomics — bigger touch targets on list rows; consider
    PWA install (add manifest) for front-desk tablets.
[ ] Keyboard shortcuts for power users (e.g. "/" focuses global search).
[ ] Accessibility pass — ARIA labels on icon-only buttons, contrast audit.
[ ] Performance — add DB indexes for common filters, query caching.
[ ] Postgres/MySQL option for tenant DBs at larger scale.
[ ] Backup monitoring dashboard (last backup time, size, success/fail).
[ ] Soft-delete / undo window for inventory deletions.
[ ] Property/unit hierarchy tree view.
[ ] Scrub leaked secrets from old git history (git filter-repo) and force
    push — coordinate with all clones before doing this.
